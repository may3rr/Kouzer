## 使用方式
1. 打开 [在线页面](https://jackielyu.is-a.dev/Kouzer/)
2. 点击「配置」，填入你的 API Key
3. 从 Word 中复制**纯文本**内容，粘贴至输入框
4. 点击「开始校对」

## 局限
1. 输入框只支持纯文本。论文中的数学公式强行粘贴会产生字符丢失，容易导致 AI 误判。
2. AI 是助理，不是法官。校对结果请视为**参考建议**，自行甄别。

## 常见问题

### 出现「请求失败：Failed to fetch」是什么原因？

这个错误是浏览器在 `fetch()` 发出请求前就将其拦截或阻断时抛出的，与服务端返回的 HTTP 状态码无关。常见原因如下：

1. **CORS 跨域限制（最常见）**
   页面托管在 `https://jackielyu.is-a.dev`（或任何静态站点），而用户配置的 API 地址来自另一个域名。浏览器的同源策略会先发送一个 OPTIONS 预检请求；若 API 服务器没有返回正确的 `Access-Control-Allow-Origin` 等 CORS 响应头，浏览器就会在请求真正发出之前将其拦截并抛出 `Failed to fetch`。
   - **解决方法**：使用明确支持跨域访问的 API 端点（如官方 DeepSeek / OpenAI 接口），或通过支持 CORS 的反向代理转发请求。

2. **API 地址填写有误**
   「接口地址」填入了格式错误的 URL（如缺少 `https://`、路径拼错、端口号错误等），导致浏览器无法建立连接。
   - **解决方法**：检查配置面板中的接口地址，确保格式正确，例如 `https://api.deepseek.com/v1/chat/completions`。

3. **网络不可用**
   本地网络断开、DNS 解析失败或防火墙阻断了对目标域名的访问，均会导致此错误。
   - **解决方法**：确认网络连接正常，并检查目标 API 域名在当前网络环境下是否可访问。

4. **HTTP / HTTPS 混合内容被阻断**
   若页面通过 `https://` 访问，而配置的 API 地址使用的是 `http://`，现代浏览器会将其视为不安全的混合内容并拦截。
   - **解决方法**：将 API 地址改为 `https://` 开头。

5. **浏览器扩展干扰**
   广告拦截器、隐私保护插件或企业安全策略扩展可能屏蔽对特定域名的请求。
   - **解决方法**：尝试在无痕模式下打开页面，或暂时禁用相关扩展后重试。
